The technical deep-dive
For developers who want to understand how it works
V
Venelin Hristov

Just now · 🌐
How I made Claude work in every IDE — with zero npm dependencies and one config file.

Here's the architecture, in plain English.

The problem: IDEs like Cursor and Antigravity hardwire their AI to their own cloud. There's no "use my own endpoint" setting. The Hyperspace proxy is running but the IDE ignores it.

The solution: run a local server that pretends to be Ollama. Every modern IDE with local AI support auto-detects Ollama. So if you fake it — they all connect.

"Continue thinks it's talking to a local Ollama server. The bridge is pretending to be Ollama. Hai is what actually returns Claude responses. Continue doesn't know or care — it just sees a working backend."
What it does technically
  • →Exposes port 11434 (Ollama API) and 11435 (OpenAI API) on 127.0.0.1 only
  • →Translates Ollama/OpenAI requests to Anthropic Messages API format
  • →Forwards to Hai at localhost:6655/anthropic/v1/messages
  • →Streams responses token-by-token — no buffering, no delay
  • →Auto-detects your API key from ~/.claude/settings.json
  • →Registers as a launchd agent — restarts on crash, starts on login

The whole bridge is one Node.js file. ~860 lines. No npm packages. Built-in http, fs, os modules only.

The installer handles everything else: detects nvm/Homebrew Node, installs Continue.dev and the required YAML extension into Cursor and Antigravity, writes ~/.continue/config.yaml.

Open source, MIT licensed.

We got the whole team using Claude in their IDEs in under 10 minutes. Here's how.

The situation: everyone on the team has Hyperspace. Claude works great in Terminal via hai proxy start. But the IDE experience was fragmented — some people on Cursor hitting quota limits, some using the native Antigravity Agent (Google's cloud), some on Zed with manual configs.

Wanted: one setup that works the same way for everyone, on any IDE, through our own Hyperspace proxy.

The solution we built: Hyperspace Bridge — a single git clone and one command per person.

Each person on the team runs
git clone https://github.com/Venelinhr/Hyperspace-Multi-IDE-Bridge.git
cd Hyperspace-Multi-IDE-Bridge && ./install.sh
What happens automatically
  • ✅Detects their Node.js — nvm, Homebrew, or system
  • ✅Finds their Hyperspace API key from ~/.claude/settings.json
  • ✅Installs Continue.dev into Cursor and Antigravity IDE automatically
  • ✅Registers a background agent that starts on login and restarts on crash
  • ✅Health check confirms everything works before the script exits

No shared server. Each person connects to their own Hai proxy. Data stays local. No per-seat cost. No vendor lock-in.

The green Hyperspace text in the IDE status bar tells everyone at a glance that they're connected.

Works with Cursor, Antigravity, Zed, JetBrains, and any Ollama-compatible tool.
