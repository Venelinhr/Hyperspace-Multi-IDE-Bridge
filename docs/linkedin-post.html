Wondering how to use Cursor, Antigravity, or Zed with Hyperspace — without a subscription?

Here's something I kept running into:
Hyperspace is running locally. Claude is right there, fast and ready.
But the moment you open your IDE — quota wall. "Upgrade to Pro." Or just silence.

So I built a fix.

Hyperspace Bridge — a tiny local server that connects your IDE to Hai, and makes Claude available in every editor, exactly the way Terminal already works.

"Your IDE is one plug shape. Hyperspace is another. The bridge is the adapter — nothing changes on either side."
Works with
  • ✅ CursorNO SUBSCRIPTION — no more "Upgrade to Pro" blockers. Claude via your own Hai proxy
  • ✅ Antigravity IDENO SUBSCRIPTION — Claude via Hyperspace, one click away in the sidebar
  • ✅ Zed — native integration, zero friction, no extension needed
  • ✅ JetBrains, Neovim, any CLI tool — if it speaks Ollama or OpenAI format, it just works

Not just VS Code. Any IDE. No subscription. No quota.

What you actually get
🌐Live web search via Tavily — "latest world news?" and Claude actually searches
💚Green Hyperspace indicator in your status bar — always know you're connected
🔒All local. No data stored. Same security path as hai proxy start
⚡Auto-detects your Hai API key — zero manual config
Install — one command
# Clone, install, done git clone https://github.com/Venelinhr/Hyperspace-Multi-IDE-Bridge.git
cd Hyperspace-Multi-IDE-Bridge
./install.sh

Detects your Node.js, finds your Hai API key automatically, installs Continue into your IDEs, registers itself as a background service, and confirms everything's working. No YAML editing. No manual proxy config.

If you use Hyperspace in Terminal and your IDE feels like a downgrade — this is the fix.
